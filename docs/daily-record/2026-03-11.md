# 2026-03-11 개발 일지

## 개요

- 에코 에이전트(dummy.py)를 대체하는 **주식 분석 특화 AI Agent** 구현.
- LangChain `@tool` 데코레이터 기반의 yfinance 도구 3종 및 LangGraph ReAct 에이전트를 설계·구현하고, 다수의 트러블슈팅을 거쳐 멀티턴 대화까지 안정화.

---

## 설계

### 아키텍처

```
사용자 메시지
  → chat.py (POST /api/v1/chat)
  → AgentService.process_query()
  → stock_agent.Agent.astream()
  → LangGraph 그래프 (model → tools → model 루프)
  → SSE 스트림 (step: model / tools / done)
  → 프론트엔드
```

### LangGraph 그래프 구조

```
[START]
  ↓
[model 노드] ── ChatResponse 호출 ──→ [execute_chat_response 노드] → [END]
  │
  └── 일반 tool 호출 ──→ [tools 노드] ──→ [model 노드] (반복)
```

- **model 노드**: LLM 호출. tool_calls 없이 텍스트로 응답 시 ChatResponse로 자동 래핑
- **tools 노드**: `get_stock_price`, `get_company_info`, `get_recent_news` 실행
- **execute_chat_response 노드**: ChatResponse tool_call에 대응하는 ToolMessage 생성 (히스토리 유효성 유지)
- **MemorySaver**: thread_id별 대화 이력 저장

### 파일 구성

| 파일 | 역할 |
|---|---|
| `app/agents/tools.py` | yfinance 기반 tool 3종 (신규) |
| `app/agents/stock_agent.py` | LangGraph ReAct 에이전트 (신규) |
| `app/agents/prompts.py` | 시스템 프롬프트 (주식 분석 역할 추가) |
| `app/services/agent_service.py` | `_create_agent()` import 변경 + 싱글턴 가드 추가 |
| `app/api/routes/chat.py` | AgentService 모듈 레벨 싱글턴으로 변경 |

## 구현

### Tool 1: `get_stock_price`
- yfinance로 현재 주가 및 전일 대비 등락률 조회
- 반환 예시: `AAPL 현재가: $260.83 | 등락률: +0.37%`
- 잘못된 티커 입력 시 명확한 에러 메시지 반환

### Tool 2: `get_company_info`
- 시가총액(조/억 단위 변환), PER, 업종 조회
- 반환 예시: `AAPL | 시가총액: 3.83조 달러 | PER: 33.02 | 업종: Technology`
- 데이터 없는 필드는 `N/A` 표시

### Tool 3: `get_recent_news`
- 최근 뉴스 최대 3건 제목 + 링크 반환
- yfinance 신버전 응답 구조(`content.title`, `content.canonicalUrl.url`) 대응
- 뉴스 없을 경우 `"관련 뉴스를 찾을 수 없습니다."` 반환

### ChatResponse
- LLM이 최종 답변을 전달할 때 호출하는 특수 tool
- `agent_service.py`가 이 tool_call을 감지해 `step: done` 이벤트로 변환

### 추가 패키지
- `yfinance` (`uv add yfinance`로 추가)

## 트러블슈팅

### 1. LLM이 ChatResponse를 호출하지 않고 텍스트로 직접 응답

**증상**
프론트엔드에 아무 답변도 표시되지 않음. 로그상 `tool_calls=[]`.

**원인**
`agent_service.py`는 `tool_calls`가 없으면 `done` 이벤트를 emit하지 않음.
LLM이 프롬프트 지시를 무시하고 텍스트로 직접 응답한 경우 처리 불가.

**해결**
1. 프롬프트에 강제 규칙 추가
   ```
   어떤 상황에서도 반드시 ChatResponse 도구를 호출하여 답변합니다.
   텍스트를 직접 반환하는 것은 금지입니다.
   ```
2. `model` 노드에 코드 레벨 fallback 추가: `tool_calls`가 없으면 응답 내용을 ChatResponse로 자동 래핑

---

### 2. 주식 무관 질문에 일반 답변 반환

**증상**
"오늘 점심 뭐 먹을까?" 질문에 메뉴 추천 답변.

**원인**
시스템 프롬프트에 도메인 제한 없음.

**해결**
프롬프트에 스코프 제한 추가:
```
주식/금융과 무관한 질문에는 답변하지 않고,
"저는 주식 분석 전용 AI입니다. 주식 관련 질문을 해주세요."라고 안내합니다.
```

### 3. 멀티턴 대화 시 컨텍스트 유실

**증상**
"삼성전자 주가 알려줘" 후 "해당 종목의 어제 주가도 알고 있어?" 질문 시 이전 대화를 기억 못함.
두 번째 요청의 `prompt_tokens`가 첫 번째와 거의 동일 (대화 이력 미포함).

**원인 1**: `AgentService`가 요청마다 새로 생성됨 → MemorySaver 초기화
- `chat.py`의 `event_generator()` 내부에서 `AgentService()` 호출

**원인 2**: `MemorySaver`를 추가해도 `_create_agent()`가 매 요청마다 새 `Agent`를 생성

**해결**
1. `chat.py`: AgentService를 모듈 레벨 싱글턴으로 이동
   ```python
   # 모듈 레벨 (서버 시작 시 1회 생성)
   agent_service = AgentService()
   ```
2. `agent_service.py`: Agent 최초 1회만 생성하도록 가드 추가
   ```python
   def _create_agent(self, thread_id=None):
       if self.agent is not None:
           return
       from app.agents.stock_agent import Agent
       self.agent = Agent()
   ```
3. `stock_agent.py`: `MemorySaver` 체크포인터 추가
   ```python
   return graph.compile(checkpointer=MemorySaver())
   ```

### 4. 멀티턴 대화 시 OpenAI 400 에러

**증상**
```
openai.BadRequestError: An assistant message with 'tool_calls' must be followed
by tool messages responding to each 'tool_call_id'.
```

**원인**
ChatResponse tool_call이 포함된 AIMessage 다음에 대응하는 ToolMessage 없이 다음 대화가 시작됨.
OpenAI API는 `tool_calls`가 있는 AIMessage 뒤에 반드시 모든 tool_call_id에 대한 ToolMessage가 있어야 함.

그래프가 ChatResponse 감지 시 바로 END로 라우팅했기 때문에 ToolMessage가 생성되지 않음.

**해결**
`execute_chat_response` 노드를 추가해 ChatResponse tool_call에 대응하는 ToolMessage를 생성:
```python
def execute_chat_response(state: MessagesState):
    call = state["messages"][-1].tool_calls[0]
    return {"messages": [ToolMessage(
        content=call["args"].get("content", ""),
        tool_call_id=call["id"],
        name="ChatResponse",
    )]}
```
- `agent_service.py`는 `"model"`, `"tools"` step만 처리하므로 이 노드 출력은 무시됨
- 대화 히스토리는 유효한 상태로 MemorySaver에 저장됨
